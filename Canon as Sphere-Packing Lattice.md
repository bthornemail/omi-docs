# Canon as Sphere-Packing Lattice

## First-Principles Construction of the OMI Canon Spine

## 0. Purpose

This document defines the 24-file OMI canon spine as a bounded construction lattice.

The point is not to claim that OMI literally is a Bravais lattice, the Leech lattice, or the Lorentzian lattice `II25,1`.

The point is to use those structures as disciplined mental models for balance, order, primitive cells, symmetry, reflection, and bounded construction.

OMI needs a way to maintain order while allowing derived declarations to grow.

The canon spine supplies that order.

The 24 root documents form the primitive canonical packing.

Derived declarations may extend, refine, quote, project, or specialize the canon, but they do not create new authority.

They must fit inside the bounded relation-space defined by the canon.

## 1. First Principle: A Canon Is Not a List

A list is only an ordering.

A canon is a bounded construction space.

A list says:

```text
this comes before that
```

A canon says:

```text
this relation may be constructed here,
under these boundaries,
with these neighboring constraints,
and with this validation path.
```

Therefore the OMI canon spine should not be understood as merely:

```text
00, 01, 02, ..., 24
```

It should be understood as a packed configuration of doctrinal cells.

Each root document is a boundary cell.

Each boundary cell has neighbors.

Each neighbor constrains interpretation.

Each interpretation constrains derived declaration.

The canon is therefore closer to a lattice or sphere packing than a table of contents.

## 2. Bravais Lattice Mental Model

A Bravais lattice begins from primitive translation vectors.

A point in the lattice is generated by integer combinations of those vectors:

```text
R = n₁a₁ + n₂a₂ + n₃a₃
```

The important idea is not the physical crystal.

The important idea is this:

```text
primitive vectors generate repeatable order
```

A lattice does not need every point to be separately invented.

Once the primitive basis is fixed, the space can be generated lawfully.

OMI uses the same design intuition.

The canon spine supplies primitive doctrinal vectors.

A derived declaration is valid only when it is generated from those primitive vectors without breaking the lattice.

In OMI terms:

```text
primitive vectors = canonical documents
unit cell = local relation basis
lattice point = derived declaration
translation = lawful projection, quotation, propagation, or carrier transformation
```

A derived declaration must not float freely.

It must be placed.

It must be addressable.

It must preserve relation identity.

It must remain reducible to the canonical basis.

## 3. Unit Cell Mental Model

A primitive unit cell is the smallest repeating unit that can reproduce the lattice.

In OMI, the local primitive cell is not a cube.

It is the tetrahedral sphere basis of the Omi-Ring.

The tetrahedral basis gives the smallest local relation structure:

```text
point
edge
face
volume
```

or, in OMI language:

```text
root
relation
witness
closure
```

This basis is local.

The 24-file canon is global.

The relation between them is:

```text
Omi-Ring tetrahedral basis = local primitive cell
24-file canon spine        = global sphere-packing boundary
```

Every derived declaration must be able to circumscribe the tetrahedral basis without tearing it away from the global canon.

That means every declaration must answer:

```text
Where is its root?
What relation does it quote?
What witness can inspect it?
What closure carries it forward?
What validation path adjudicates it?
```

## 4. N-Ball and N-Sphere Interpretation

An n-ball is the bounded interior.

An n-sphere is the boundary surface.

For OMI:

```text
n-ball   = allowed construction interior
n-sphere = canonical boundary of interpretation
```

The 24-file canon spine is the n-sphere boundary.

It defines what the construction space is allowed to touch.

Derived declarations live in the n-ball interior.

They may move, refine, and compose inside the boundary, but they may not redefine the boundary.

Therefore:

```text
canon files = boundary spheres
derived declarations = interior construction balls
validation = test of lawful placement
projection = visible surface after acceptance
```

A derived declaration that cannot be placed inside the bounded canon is not accepted OMI state.

It may remain a candidate.

It may remain a note.

It may remain an external carrier artifact.

But it is not canonically accepted state.

## 5. Sphere Packing as Canon Discipline

Sphere packing asks how equal or structured spheres can be arranged without overlap while preserving density, symmetry, and contact.

OMI uses this as a mental model for doctrinal balance.

Each canonical document is a sphere-cell.

It must touch neighboring cells without collapsing into them.

For example:

```text
OMI-LISP touches BLACKBOARD,
but OMI-LISP is not the BLACKBOARD.

BLACKBOARD touches NOTATION-MULTIPLEXING,
but BLACKBOARD is not the carrier.

PROJECTION-FACES touch TRANSFORM-CARRIERS,
but projection is not authority.

CANONICAL-CONSTITUTION touches MASTER-ARCHITECTURE,
but architecture is not law.
```

The packing rule is:

```text
touching is allowed
overlap of authority is not allowed
```

This prevents category errors.

It keeps grammar from becoming validation.

It keeps projection from becoming acceptance.

It keeps carriers from becoming identity.

It keeps implementation from becoming doctrine.

## 7. Relation to the Leech Lattice Mental Model

The Leech lattice is useful here as a mental model because it is a highly symmetric 24-dimensional lattice with exceptional packing properties.

OMI should not claim to be the Leech lattice.

Instead, OMI can borrow the discipline:

```text
24-dimensional balance
high symmetry
no unnecessary roots
dense non-overlap
deep and shallow hole awareness
```

In OMI language:

```text
24 dimensions of canon
balanced doctrinal cells
no arbitrary authority roots
dense relation coverage without overlap
awareness of gaps, holes, and unresolved declarations
```

A “hole” in this mental model is not a mathematical Leech hole inside OMI.

It is an architectural gap:

```text
a declaration that almost fits,
but lacks a lawful neighbor,
witness,
closure,
or validation path.
```

Deep holes are major missing doctrines.

Shallow holes are local missing definitions.

The canon’s job is not to eliminate all future holes.

The canon’s job is to make holes visible and classifiable.

The Omi-Ring supplies the local primitive basis.

## 10. Karnaugh Maps as Adjudication Surface

A Karnaugh map is useful in OMI as a deterministic reduction surface.

It does not create truth.

It organizes candidate truth conditions so they can be reduced.

In OMI:

```text
candidate declaration
→ truth gate
→ decision table
→ Karnaugh reduction
→ Tetragrammatron adjudication
```

The Karnaugh map is therefore not authority.

It is a reduction surface.

The Tetragrammatron adjudicates over the reduced relation.

The validation result determines whether the relation becomes accepted Omi-State.

Therefore:

```text
Karnaugh maps reduce.
Tetragrammatron adjudicates.
Validation determines.
Attestation witnesses.
Projection displays.
```

## 11. Derived Declaration Rule

A derived declaration is valid only if it can be placed in the canon packing.

Placement requires:

```text
1. It has a canonical root.
2. It quotes a relation rather than asserting authority.
3. It preserves place-value identity.
4. It declares its carrier face if carried.
5. It exposes a witness path.
6. It can be reduced by deterministic rules.
7. It can be adjudicated by the Tetragrammatron.
8. It can be recorded as accepted Omi-State only after validation.
9. It can be projected only after acceptance.
```

If any of these fail, the declaration remains unresolved.

Unresolved does not mean false.

Unresolved means:

```text
not yet placed in the packing
```

## 12. Balance and Order

The reason to use Bravais and Leech lattice mental models is practical.

They teach:

```text
do not place points arbitrarily
choose primitive basis carefully
preserve symmetry
track neighbors
avoid overlap
make holes visible
distinguish primitive cells from convenient cells
distinguish boundary from interior
```

That is exactly what the canon needs.

OMI should remain balanced by asking, for each proposed change:

```text
Is this a primitive vector or a derived translation?
Is this a unit cell or a conventional convenience?
Is this a boundary document or an interior declaration?
Is this a relation, a projection, a carrier, or an authority claim?
Does it preserve the tetrahedral Omi-Ring basis?
Can the Tetragrammatron adjudicate it?
```
